系统的安全哲学

这套系统有一条贯穿始终的原则：自动化是"变化提醒"，不是"攻击武器"。

✅
默认只做被动收集

subfinder、dnsx、httpx、低风险 Nuclei——这些都是"看"，不是"打"

🚫
默认排除危险操作

DoS、brute-force、intrusive、RCE、LFI、SQLi 相关的 Nuclei 模板全部排除

🔐
必须先确认授权

scope 配置里 legal_ack=true 才能运行 pipeline，这是硬性门槛

👁️
人工验证是必须的最后一步

所有自动化输出只是"候选"，报告前必须手工确认真实影响

⚠️
为什么这很重要？

Bug Bounty 是在授权范围内测试。越界测试不只是违规——它可能触犯法律。这套系统的每个默认值都在保护你，不要绕过它们。

组件之间如何沟通

当 VPS 跑完一次侦察，下面是各组件之间发生的对话：

R
Recon Pipeline

发现了！api.v2.target.com 是新出现的子域名，今天之前没有这个域名。

S
Scope Guard

等等，先让我检查。api.v2.target.com 匹配 *.target.com，在授权白名单里，不在排除列表里。放行！

D
Diff Engine

对比上次运行记录，确认这是新资产。生成 diff 报告，标记为 "+ api.v2.target.com"。

N
Notify

发送 Discord 通知：只发 diff 摘要（新增1个子域名），不发任何 token、cookie 或敏感响应体。

你
你（猎人）

收到通知，打开 Burp Suite，手工测试这个新 API 端点——这才是真正的"找漏洞"开始的地方。
