为什么情报这么难收集？

想象你订阅了 100 份学术期刊，每天新增 200 篇论文——但你每天只有 30 分钟阅读时间。你会错过绝大多数有价值的内容。

这就是 Bug Bounty 情报的现状。X 上每天有几十条有价值的猎人帖子，HackerOne 每月发布数百份公开报告，PortSwigger 研究博客持续产出新技术——根本看不完。

🔑

💡
Token 是什么？

Token 是 AI 模型处理文本的基本单位，也是费用的计量单位。这个提示词被精心设计，让 Grok 跳过无用内容，节省你的 API 费用。

一张技巧卡片有什么

情报经过 Grok 提炼后，变成标准化的"技巧卡片"存进知识库。每张卡都有固定的字段：

标题 简短的技巧名称，如"invoice_id IDOR via export endpoint"
漏洞类型 IDOR、OAuth bypass、GraphQL mass assignment 等分类
前置条件 需要两个账号？需要付费订阅？需要特定 API 版本？
核心 trick 具体操作步骤，可以直接用于测试
可迁移目标 这个 trick 最可能在哪类平台上奏效
来源 URL 每条声明必须有来源——这是系统的质量底线
置信度 / 风险等级 high/medium/low；low/medium/high——帮你决定优先测哪个
🔬
关键工程决策

知识库里的每条记录都有来源 URL。grok_kb_validate.py 会拒绝任何没有来源的卡片。这防止了 AI 幻觉：Grok 生成内容，必须能指向真实的来源才算数。
