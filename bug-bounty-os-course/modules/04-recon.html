04

自动化侦察引擎

用 VPS 每天静默扫描——只在"有变化"时才叫醒你

它是一个永不睡觉的助手

想象你雇了一个助手，他的工作是：每天凌晨3点去目标公司门口，记录门口有几辆车、多少员工进出、有没有新开的侧门——然后只有当今天和昨天不一样时，才来敲你的门。

这就是 recon_pipeline.py 的工作模式。它在你睡觉的时候运行，发现变化时才通过 Discord 或 Telegram 通知你。

🖥️
什么是 VPS？

VPS 是你在互联网上租的一台永远开着的服务器，可以7×24小时运行脚本。

Pipeline 的7个步骤

每次运行 recon_pipeline.py，按这个顺序执行：

1
检查授权（legal_ack 守门）

读取 scope 配置，legal_ack=false 时直接退出，拒绝运行。这是第一道护栏。

2
随机延迟启动

随机等待1-4秒再开始，避免每次都在整点触发，减少被目标检测到的风险

3
被动子域名收集（subfinder）

subfinder 从公开数据源（证书透明度日志、DNS 数据库）找出所有子域名，完全被动

4
Scope 过滤（scope_guard）

每个发现的子域名都过 scope_guard.py，不在白名单里的全部丢弃

5
DNS 解析（dnsx）

DNS 解析确认哪些子域名实际上有服务器在响应，过滤掉已失效的域名

6
HTTP 探活（httpx）

httpx 探测哪些资产有 Web 服务，收集页面标题、技术栈信息，速率限制25次/秒

7
低风险 Nuclei 扫描

只跑 exposure、misconfig、takeover、tech 标签，排除所有 dos/rce/sqli/exploit 相关模板

scope 配置文件如何控制一切

整个 pipeline 的行为由一个 JSON 配置文件控制。这是真实的 scope.example.json 里的关键部分：

CODE 
{
  "legal_ack": false,
  "in_scope_domains": [
    "example.com",
    "*.example.com"
  ],
  "out_of_scope_domains": [
    "*.internal.example.com"
  ],
  "recon": {
    "active_crawl": false,
    "nuclei_low_risk": true
  },
  "limits": {
    "httpx_rate": 25
  }
}
PLAIN ENGLISH

开始这个大括号里的配置

false = 不运行！只有你确认授权后才改成 true

允许测试的域名列表开始

example.com 本身可以测试

所有 *.example.com 子域名（如 api.example.com）都可以

绝对不能碰的域名列表

internal 子域名是内部系统，不能碰

侦察行为设置

主动爬虫关闭——只被动收集，不主动爬取页面

低风险 Nuclei 扫描开启

速率限制设置

httpx 每秒最多发25个请求，不轰炸目标服务器

scope_guard.py 如何过滤域名

这是 scope_guard.py 里最关键的逻辑——只有30行，但它是整个系统的安全底线：

CODE 
def is_allowed(host: str, cfg: dict) -> bool:
    ins = cfg.get('in_scope_domains', [])
    outs = cfg.get('out_of_scope_domains', [])
    if not any(host_matches(host, p) for p in ins):
        return False
    if any(host_matches(host, p) for p in outs):
        return False
    return True
PLAIN ENGLISH

定义一个函数：给定域名和配置，判断能不能测

读取配置里的白名单域名列表

读取配置里的黑名单（排除）域名列表

如果这个域名不在白名单里的任何一条记录里...

...立刻返回"不允许"（宁可漏掉，也不越界）

如果这个域名匹配黑名单里的任何一条记录...

...也返回"不允许"（明确禁止就是禁止）

两道检查都过了，才返回"允许"

💡
"双重否定"设计

注意这个函数先检查"是否在白名单"，再检查"是否在黑名单"。这是"默认拒绝"原则——不明确允许的域名，一律不测。这比"只检查黑名单"安全得多，因为你不可能事先列出所有不该测的域名。

你设置了每天凌晨3点跑一次 recon_pipeline.py。假设目标公司三天没有任何变化，你会收到几次 Discord 通知？
